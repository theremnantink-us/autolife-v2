## Context

Фаза 0 закрыла дыры безопасности, но не решила стратегическую задачу: **что вообще такое AutoLife Detail** — как бренд, где он стоит относительно конкурентов, и какой визуальный + речевой язык должны использовать редизайн (Phase 2), реклама (Phase 6) и маркетинговая стратегия (Phase 8).

Без brand book эти фазы будут угадыванием: цвета — «какие-то красивые», тон в Яндекс.Директе — «как у всех», фото в Nano Banana — «в стиле машинной рекламы». С brand book каждый downstream-актив ссылается на одни правила, бренд становится узнаваем.

**Стратегия:** Refresh (не rebrand). SEO-история домена `autolife-detail.ru` сохраняется, имя и dark-эстетика остаются. Меняются: зафиксированная палитра (chrome-on-black вместо хаотичного burgundy), логотип (получен от клиента), типографика (Michroma для wordmark + Exo 2 для всего), формализованные positioning / tone of voice / tagline.

**Важное решение по ответственности:** бренд-язык намеренно избегает классовых qualifier'ов («представительский», «люкс») и обещаний-гарантий результата («стандарт производителя», «документированный результат»). Это снижает юридическую и репутационную ответственность бизнеса при сохранении премиум-восприятия через визуал и фактуру материалов.

| 3 | [Brooklands Detailing](https://brooklandsdetailing.ru/) | Звенигородское 28, Москва | по запросу | Сильнейший бренд-recognition в сегменте, «с 2014», «1 из 20 проходит отбор». Слабое место — Zoon 3.5/5 (операционка), центр города (далеко от Рублёвки) |

**Второстепенные:** Rublevka Detailing (orgs.biz, слабая диджитал-репутация), Royal Detailing (Куркино, слабый бренд-голос), Mono Detailing (цены «от 2500» режут премиум-восприятие).

**Где AutoLife может выиграть:** спокойный локальный премиум без lifestyle-шоу (Unlim) и без блогерского tilt (Brooklands) — «ваш районный детейлинг-центр». Visual: chrome-silver вместо gold (Unlim/Royal) и вместо burgundy (старый AutoLife CSS), выделяется без ухода из категории.

## Brand identity

### Имя и домен

- **Полное имя (официальное):** AutoLife Detail / АвтоЛайф Детейл
- **Сокращённое (UI, разговорное):** AutoLife / АвтоЛайф
- **Домен:** `autolife-detail.ru` (единственный живой)
- **Расположение:** Рублёво-Успенское шоссе, Москва

### Positioning statement

> AutoLife Detail — детейлинг-центр на Рублёво-Успенском шоссе. Работаем с профессиональными материалами для ухода за автомобилем.

Намеренно короткое, без классовых qualifier'ов и обещаний результата. Позиция создаётся визуалом, локацией и фактурой — не декларациями.

### Tagline

> **«Точность. Материалы. Результат.»**

Триада без глагола: ритм, сухость, premium-серьёзность. Не обещание ("вернём", "гарантируем"), а имплицитная структура сервиса — три опоры.

### Целевой клиент

Владельцы Bentley / Mercedes / Porsche / Range Rover / Maybach, постоянно проживающие на Рублёвке или Новой Риге. Ценят: регулярность обслуживания с одним и тем же мастером, низкий шум (без lifestyle-шоу), профессиональные материалы, предсказуемость сроков.

**Чего клиент НЕ ценит:** скидочный шиномонтаж за 450₽, блогерский лоск, теннис при автосервисе, эмодзи в SMS, обращение на «ты».

## Tone of voice

### Правила

1. **Обращение — «вы»** (строчная), кроме официальных писем, где «Вы» (прописная).
2. **Без эмодзи** в customer-facing коммуникации (SMS, email, onboarding). Допустимо в соцсетях умеренно (1 эмодзи на пост максимум).
3. **Без классовых qualifier'ов:** не «представительский», не «люкс-сегмент», не «премиум-уровень». Премиум показывается материалами и визуалом, не словами.
4. **Без обещаний результата:** не «вернём заводской стандарт», не «гарантируем идеальную кожу», не «результат документируется». Только фактура процесса: «двухэтапная полировка», «подготовка 8 часов», «материалы Koch Chemie».
5. **Числа и факты вместо прилагательных:** «16 часов работы» > «тщательно и аккуратно».
6. **Глагол в первом лице множественного:** «работаем», «используем», «предлагаем». Не «я» и не «команда мастеров AutoLife».
7. **Без superlatives:** не «лучший», не «единственный», не «уникальный». «Один из», «наш» — ок.

### Примеры

| Контекст | ✓ Так | ✗ Не так |
|---|---|---|
| Hero subtitle | «Полировка, керамика, химчистка, восстановление хрома. Работаем с профессиональными материалами.» | «Премиальный детейлинг высочайшего уровня для автомобилей представительского класса» |
| Карточка услуги (керамика) | «Керамическое покрытие последнего поколения. Подготовка — до 8 часов. Гарантия — 12 месяцев.» | «Гарантированно лучшее керамическое покрытие, которое защитит ваш автомобиль навсегда» |
| Подтверждение записи (SMS) | «Ваш автомобиль ожидается к 14:30. При изменении планов просим уведомить заранее.» | «🚗 Спасибо за запись! Ждём вас в назначенное время ✨» |
| Ошибка формы | «К сожалению, выбранное время уже занято. Ближайшее свободное — 16 апреля, 11:00.» | «Ой! Что-то пошло не так. Попробуйте ещё раз!» |
| Пост в соцсети | «Двухэтапная полировка Mercedes-Maybach S680, 16 часов работы. Материалы Koch Chemie и P&S.» | «Завершили невероятную работу над этим красавцем! Наши мастера — настоящие волшебники 💫» |

## Visual identity

### Logo / Wordmark

**Лого (brand-mark):** предоставлен клиентом — силуэт спорт-купе в chrome-градиенте над wordmark «AUTOLIFE». Файл должен быть экспортирован в форматах:

- `logo-full.svg` — вектор для responsive использования (масштабируется без потерь)
- `logo-full@1x.png`, `logo-full@2x.png` — растр для legacy поддержки (email, соцсети)
- `logo-mark.svg` — только силуэт авто без wordmark (для favicon, аватарок)
- `logo-wordmark.svg` — только «AUTOLIFE» текст (для случаев без силуэта)

**TODO для клиента:** прислать исходный логотип в SVG или AI. Текущий PNG (получен в чате) не подходит для production — артефакты при скейле.

**Wordmark (type-рендер):**

```css
font-family: 'Michroma', sans-serif;
font-weight: 400; /* Michroma only has Regular */
letter-spacing: 0.12em;
-webkit-text-stroke: 0.7px currentColor;
paint-order: stroke fill;
background: linear-gradient(180deg, #f5f5f5 0%, #8a8a8a 50%, #4a4a4a 100%);
-webkit-background-clip: text;
background-clip: text;
-webkit-text-fill-color: transparent;
```

Применение: только для H1 на hero и для ре-рендера логотипа в DOM (там где нужен текст, не изображение). Не для H2-H6 и не для body.

### Palette — Chrome on Black

Все токены определяются в `:root` CSS как custom properties:

| Токен | Hex | Применение |
|---|---|---|
| `--bg` | `#0a0a0a` | Основной фон body |
| `--bg-surface` | `#141414` | Карточки, секции над фоном |
| `--bg-elevated` | `#1c1c1c` | Модалки, dropdowns, hovered surface |
| `--border` | `#222222` | Границы между секциями |
| `--border-ring` | `rgba(255,255,255,0.12)` | Активные состояния (focus ring, selected) |
| `--chrome-gradient` | `linear-gradient(180deg, #f5f5f5 0%, #8a8a8a 50%, #4a4a4a 100%)` | Только wordmark + главные CTA (редко) |
| `--silver` | `#d8d8d8` | Вторичные акценты, иконки, линии |
| `--text` | `#f0f0f0` | Основной текст |
| `--text-muted` | `#888888` | Вторичный текст, metadata, label |
| `--text-subtle` | `#5a5a5a` | Disabled, placeholder |

**Функциональные цвета** (минимально, только toast/validation, не в brand-visual):

| Токен | Hex | Где |
|---|---|---|
| `--success` | `#9ca77c` | Toast «запись создана» |
| `--error` | `#b54747` | Form validation errors |
| `--warning` | `#c9a961` | Пред-действия, напоминания |

**Критично:** никакого golden/bronze/burgundy в brand-идентите. Пара silver + black + white — и всё. Функциональные акценты используются утилитарно.

### Typography

| Роль | Шрифт | Вес | Размер | Применение |
|---|---|---|---|---|
| Wordmark | Michroma | 400 + 0.7px stroke | 56-60px hero, 36-40px header | Логотип, H1 на hero |
| H1 display | Exo 2 | 700 | 42px / line-height 1.1 / letter-spacing -0.01em | Главные заголовки секций |
| H2 section | Exo 2 | 600 | 28px / 1.2 | Заголовки подсекций |
| H3 card | Exo 2 | 600 | 18px | Карточки услуг |
| Body | Exo 2 | 400 | 15px / 1.7 / color #c0c0c0 | Основной текст |
| Label / Meta | Exo 2 | 500 | 11px / letter-spacing 3px / uppercase / #888 | Категории, мета-теги |

**Подключение:**

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Michroma&family=Chakra+Petch:wght@400;500;600;700&display=swap" rel="stylesheet">
```

**Старые шрифты к удалению:** Inter, Montserrat, Orbitron — уже были подключены, но замещаются Exo 2. В Phase 2 (редизайн) — cleanup fonts.googleapis.com строк в `<head>`.

## Phase 0 domain patch (prerequisite for Phase 2 deploy)

**Файлы для правки:**

- `public_html/.env.example` — `CORS_ORIGIN=https://autolife-detail.ru`, `APP_URL=https://autolife-detail.ru`
- `public_html/config.php` — default для `CORS_ORIGIN` → `https://autolife-detail.ru`
- `public_html/.htaccess` — CSP `frame-src` если ссылается на `autolife.ru` (сейчас нет, но проверить при доливке Яндекс.Метрики в Phase 7)
- `docs/superpowers/specs/2026-04-24-security-hotfix-design.md` — обновить примеры curl-верификации на `autolife-detail.ru`

**Реализация:** отдельный коммит на ветке `phase-0/security-hotfix` с сообщением `fix(security): correct domain reference to autolife-detail.ru`. Не требует ревью — text-only замена.
